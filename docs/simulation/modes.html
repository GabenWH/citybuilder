Simulation Modes

Mode-specific rules and how they connect through transfers.

Overview | Backend API | Modes

Shared Concepts

  • Mode networks are separate graphs with their own rules.
  • Transfers connect modes (walk-to-stop, station, airport).
  • Routing can be mode-specific or multi-modal.

Road (Car + Bus)

  • Lane-cell flow model for scale.
  • Intersection phases or priority rules.
  • Bus stops are lane-side transfer nodes.

Rail (Train + Metro)

  • Schedule-based movement per line.
  • Capacity constraints at stations.
  • Transfers to walk grid or bus stops.

Air (Planes)

  • OD between airports with coarse travel times.
  • Flights treated as long-duration trips.
  • Transfers via airport nodes.

Walking

  • Coarse grid or sidewalk graph.
  • Low cost, short distance travel.
  • Primary glue between other modes.

Mode Choice (v1)

Simple static weights for now. Future: dynamic pricing, congestion, time-of-day.

mode = WeightedChoice({

  Car: 0.5,
  Bus: 0.15,
